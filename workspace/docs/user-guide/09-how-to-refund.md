# 如何處理退款（3 種場景）

## 你想做什麼

客戶要退款、或供應商退錢給我們。先判斷是哪種場景、再走對應流程。

## 場景判斷表

| 場景 | 觸發點 | 流程 |
|---|---|---|
| **A. 客戶取消、未收款** | 訂單還沒收任何錢、客戶取消 | 訂單取消即可（最簡單） |
| **B. 已收客戶款、要退** | 客戶已付、現在要退錢給客戶 | 開「退款收款」（負金額）|
| **C. 已付供應商、對方退** | 我們付了機票 / 飯店、供應商退錢 | 開「退款請款」（負金額）|

## A. 客戶取消、未收款

1. 進「客戶管理 → 訂單列表」
2. 點該訂單 → 「取消訂單」
3. 系統自動更新訂單狀態為「已取消」
4. 不用做任何財務動作（沒收錢、不用退）

## B. 已收客戶款 → 退款（最常用）

1. 進「財務管理 → 收款管理」
2. 按「新增收款」
3. 選同一個客戶 + 同一個訂單 / 團
4. **金額填負數**（譬如原本收 30,000、退款 10,000 → 填 -10,000）
5. **付款方式選「退款匯款」/「退款現金」等對應**
6. 備註寫「客戶取消、退一半」之類
7. 儲存 → 主管核准 → 出納實際退錢給客戶 → 標「已退款」
8. 系統自動產傳票（借：銀行存款負 / 貸：應收帳款負）

## C. 已付供應商 → 對方退錢

1. 進「財務管理 → 請款管理」
2. 按「新增請款」
3. 選同一個供應商
4. **金額填負數**（譬如原本付 50,000、供應商退 5,000 → 填 -5,000）
5. **請款類別維持原本的**（譬如「機票」就還是機票）
6. 備註寫「機票取消供應商退款」
7. 儲存 → 主管核准 → 系統自動入帳

## 常見問題

**Q：為什麼用負金額、不開「退款單」這種獨立功能？**
A：會計上「退款」就是「收款 / 請款」的反向、用同一機制處理乾淨、不用另開表。

**Q：客戶要分期退（譬如分 3 期退）怎做？**
A：每期開一筆負金額收款、備註「退款 1/3 期」「2/3 期」、系統會自動加總。

**Q：客戶不接受退款、要轉移到下次團、能直接「收款轉移」嗎？**
A：可以。不用退錢、直接走「收款轉移」到下次團即可。比退款省事。

**Q：退款給客戶要不要扣手續費？**
A：依公司政策。如果要扣、開兩筆：
- 退客戶的金額（負）
- 手續費請款（正、付給銀行）

**Q：客訴賠償怎麼歸類？**
A：開「請款」、類別選「客訴賠償」（沒有的話到「財務 → 設定 → 公司支出項目」加）。

## 對應 entry points

- **退款收款**（場景 B）：`/finance/payments` → 新增收款（負金額）
- **退款請款**（場景 C）：`/finance/requests` → 新增請款（負金額）
- **訂單取消**（場景 A）：訂單列表點選後取消
