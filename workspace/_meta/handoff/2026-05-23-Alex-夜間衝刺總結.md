## 7. 已知技術債 / 風險

| 風險                                                                         | 影響                                             | 緩解                               |
| ---------------------------------------------------------------------------- | ------------------------------------------------ | ---------------------------------- |
| `as never` cast 散在 supabase insert（payload jsonb / dynamic table）        | 型別安全降低                                     | Phase 2 改用 type-safe wrapper     |
| 共用對話元件建好但舊 UI 未換                                                 | 視覺仍可能不一致                                 | Phase 2 漸進 migrate、新功能優先用 |
| Mock pay flow 沒有客戶端 abuse 防護（譬如可重複 POST webhook 一直 captured） | mock 階段問題小、Phase 2 接真實 webhook 後不存在 | Phase 2 用永豐 nonce + sign 驗證   |
| approval_requests 沒有 expires 自動清                                        | 過期請求殘留                                     | 後續加 cron job                    |
| Supabase Pro plan branching 沒開、PR CI 只做 code-only sanity                | 動 schema 仍可能撞 production                    | 短期接受、需要時升 Pro             |
