## SSOT 2 — Capabilities（modules/ → capabilities.ts）

**機制**：`codegen:permissions` 從 modules/ 衍生到 `src/lib/permissions/capabilities.ts`

**audit:rls 結果**：✅ 守住（DRY、無游離字串）

**⚠️ drift 發現**：audit:rls 報告 10 capability 在 capabilities.ts 有、但 modules/ 衍生沒：

```
channels.manage
facebook_bot.config / .write
finance.advance_payment.write
instagram_bot.config / .write
line_bot.config / .write
```

**判定**：這些 drift 是 3 個已廢 bot module（facebook_bot / instagram_bot / line_bot）的殘留 capability。**不影響資安**（code 層無 caller），但會造成混淆。建議清理。

**分析**：以下條件同時滿足才會觸發同類事故：

1. DB trigger 建了 branches（HQ）
2. API create-tenant-seed.ts 也建 branches（HQ）

**現況**：

- trigger 已砍（20260517970000）

**結論**：不會。使用相同模組的工程師若未跑 audit:writes 仍可能觸發、但已比 5/12 前提早阻止。
