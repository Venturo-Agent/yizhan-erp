# 顧客建檔流程 — 設計提案

> 日期：2026-05-26
> 拍板：William
> 狀態：規格定案、待 fix-safe 實作
> 相關事故：福岡團 FUK260707A 一天內生出 320 張空白重複顧客卡

---

## 一、問題（為什麼要重設計）

### 現象

福岡團（FUK260707A，76 人）在 2026-05-26 02:59–03:03 短短 4 分鐘內，
產生了 **320 張空白顧客卡**（每個名字最多 9 張重複、全部無護照/身分證/生日、全未驗證）。
團裡 54 個團員連到垃圾卡、22 個沒連、**0 個連到正確的本尊卡**。

### 根因

**把「團員名單」和「顧客主檔」當成同一件事處理，而且用錯了觸發點與比對鍵。**

1. **觸發點錯**：`useEditModeSyncCustomers` 在「關閉編輯模式」時，
   把每個「有名字、沒連顧客」的團員**默默自動建成顧客**。沒確認、沒提醒。
2. **比對鍵錯**：建之前只用「護照號／身分證號」查重複，**完全不看名字**。
   貼進來的名單只有名字、沒證件 → 每次都查無 → 每次都建新卡。
3. **連本尊都找不到**：DB 裡明明有本尊（C000244 等），但比對要證件、不看名字 → 比不到。
4. **反覆測試 = 反覆生**：每貼一輪名單、關一次編輯模式，就把全團重建一批 → 累積 320 張。

### 概念釐清

| | 團員名單（order_members） | 顧客主檔（customers） |
|---|---|---|
| 本質 | 某團某單的一行、進行中的草稿 | 可重複使用的聯絡人、有身分、有歷史 |
| 常態 | 常常先只有名字、護照晚點補 | 確認過身分的正式檔案 |
| 該不該急著建 | — | **不該**：名單會有一堆「只有名字、還沒成形」的人 |

---

## 二、核心原則

> **「建顧客」要同時滿足：有身分 + 有意圖。**
> 打名字只進名單；要變顧客是「明確動作 + 有身分證件」那一刻。

---

## 三、正確流程 v1（定案）

### 1. 打/貼名字 → 只進名單、不碰顧客
團員可以一直只有名字。**拿掉「關閉編輯模式自動建顧客」**（元兇）。
關閉編輯模式 = 純粹關閉，對顧客零動作。

### 2. 建/連顧客的兩個明確入口（都需有護照或身分證號）

**(A) 單筆 → ⚠️ 驗證**（沿用現有流程）
個別點某團員的 ⚠️ → 驗證對話框 → 確認時比對既有顧客、撞名跳通知。
適用零星處理個別人。

**(B) 整團 → 「比對顧客」批次按鈕**（新增、放成員工具列）
貼完整團、想建檔時自己按。挑「有護照或身分證、還沒連顧客」的人一次比對，
出一張審核清單讓使用者一次處理。適用貼一整團（尤其多為舊客人時）。

> 貼上本身**不自動觸發**比對（避免驚嚇 + 避免每次關閉重跑）；由 (A)/(B) 明確觸發。

### 3. 比對規則（關鍵：護照會換、身分證不會換）

| 比對鍵 | 角色 | 比中時的動作 |
|---|---|---|
| **身分證號** | 「同一人」的錨點（一輩子不變、主鍵）| = 同一人 → 連結 + 更新其資料（含換新護照）|
| **護照號** | **會變動的資料**、不當身分錨點 | 比中可參考、但主要拿來「更新」 |
| **名字** | 弱線索 | 撞名但身分證沒比中 → **跳通知問**「是同一人嗎？」→ 更新本尊 / 建新 |
| 都沒比中 | — | 建新顧客 |

**外國人/沒身分證**：改用「護照號 + 名字」，撞名一樣跳通知（不只認護照，因護照會換）。

## 四、批次「比對顧客」審核清單（UI 設計）

按下「比對顧客」後，出一張清單，每個團員一列：

```
比對結果（70 筆）                          [全部套用] [取消]
─────────────────────────────────────────────
✓ 王小明   → 對到舊客 C000123（身分證相符）   自動連結
✓ 李大華   → 對到舊客 C000088（身分證相符）   自動連結
⚠ 林聖芬   → 找到 2 個同名，請選 ▾           [C000252 / C000301 / 建新]
⚠ 陳美華   → 同名但資料不同，是同一人嗎？     [更新 / 建新]
＋ 周新客   → 查無 → 將建立新顧客
─────────────────────────────────────────────
```

- **✓ 身分證相符** → 自動連結（無需人工）
- **⚠ 撞名 / 資料不符** → 使用者逐筆決定（連既有 / 更新 / 建新 / 略過）
- **＋ 查無** → 將建新顧客
- 使用者看一眼、處理問號、按「全部套用」→ 一次連好/建好整團

> William 的真實名單通常是「名字 + 身分證號」→ 大部分走 ✓ 自動連結、少數要確認。

## 五、要改的程式（fix-safe、逐項拍板）

| # | 檔案 | 改什麼 | 大小 |
|---|---|---|---|
| 1 | `_hooks/useEditModeSyncCustomers.ts` | **整段拿掉自動建顧客**，保留編輯模式開關本身 | 小 |
| 2 | `_hooks/useMemberEditDialog.ts`（handleSaveEdit）| 有護照/身分證才建；比對改「身分證錨點 → 名字」；撞名不自動建、改觸發通知 | 中 |
| 3 | 新增「撞名確認對話框」 | 「找到既有顧客 X，是同一人嗎？」→ 更新既有 / 建新 | 中（新 UI）|
| 4 | 新增「比對顧客」批次流程 | 工具列按鈕 + 審核清單對話框 + 批次比對/連結/建立邏輯 | 大（新 UI + 邏輯）|
| 5 | `_hooks/passport/usePassportValidation.ts`（OCR）| **改為只填名單資料、不建顧客**（顧客一律等驗證/比對才建）| 小–中 |

### 比對邏輯應集中（不要散刻）
建議抽一支共用比對函式（input：name/national_id/passport → output：match 結果分類），
給單筆驗證 (A) 與批次比對 (B) 共用，避免兩邊規則不一致。
