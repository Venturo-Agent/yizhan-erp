## Phase 2 動工順序（拿到 UAT 特店號後）

```
Day 1-2  測試環境建置
  - 本機 supabase start
  - .env.development.local 範本 + seed script
  - Supabase branch CI workflow

Day 3-4  sinopac module 骨架
  - src/lib/payment-providers/sinopac/
    ├── config.ts       (env 切換：mock / uat / production)
    ├── crypto.ts       (HashID / Sign / AES-CBC、搬 line-payment-bot)
    ├── epos.ts         (URL 付款頁組裝、TransMode=0)
    └── webhook.ts      (永豐通知冪等處理)

Day 5    Webhook 接入
  - /api/payment-webhooks/sinopac/epos/notify (取代 /mock/[token])
  - 冪等：oid + TransNo 為去重 key、5 秒內回 200
  - 寫入 payment_transactions + receipts.status

Day 6    Client 頁面切換
  - /pay/mock/[token] → /pay/sinopac/[token]
  - iframe 嵌 epos.sinopac.com / eposuat.sinopac.com
  - 接收 3D 跳轉 / 前台通知 redirect

Day 7    End-to-end 測試
  - 本機 → UAT 特店號 → 測試卡 4311 9500 0000 0000
  - 從新增收款 row 產連結 → 客戶刷卡 → webhook → receipt confirmed
  - 30 次 dry-run 看穩定性（Alex 紀律：production 前 30 次測試）
```
