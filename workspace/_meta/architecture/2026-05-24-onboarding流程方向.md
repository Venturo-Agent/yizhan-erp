# Onboarding 流程方向（William 2026-05-24 討論 + 主 Claude 建議）

> 起因：改密碼已是強制第一頁、登入後該先做什麼？印章/發票要不要一開始強制？教學內容也跟著定。
> 連動：剛移除的 SetupBanner（全域設定牆）。

## William 的核心問題
1. 登入 → 強制改密碼（已有）→ 然後呢？
2. 不可能每件事都老闆做 —— 老闆帳號開了、但要設印章/發票等、誰做？
3. 印章/發票一開始強制會很煩、但「沒印章不能用」？

## 主 Claude 建議（business 角度）

### 1. 老闆第一步 = 建員工 / 角色（人資），不是設定牆
登入 → 改密碼 → 落在有用的首頁 + 第一個引導「建立員工 + 派角色權限」。
理由：老闆先建團隊 + 委派、後面細設交給對的人（會計設印章/發票、業務開團）。解「不可能每件事老闆做」。

### 2. 印章 / 發票 = 用到時才設（just-in-time），不強制一開始
- **澄清「沒印章不能用」**：只有「合約 / 正式文件」需要公司印章。建團 / 開團 / 訂單 / 客戶 / 收款 全都不用印章。所以印章不是「整個系統的門檻」、只是「合約功能的門檻」。
- 做法：要產合約 / 蓋章文件時、才提示「此功能需要公司印章、去設定」、由有 capability 的人（會計/admin）設。不是一開始擋住老闆。
- 好處：第一次登入體驗順、不是一面煩人的設定牆；設定發生在「真的需要的那一刻」、由對的角色做。

### 3. 發票特別注意 ⚠️
`travel_invoice` 模組目前**凍住**（Phase 2、8 月後）。所以「字軌號碼 / 發票章」**現在根本不該出現在 onboarding**。
（這正是健檢抓到的 P0：TenantPrepSection 還在叫客戶準備發票章、但模組凍了、客戶傳了沒地方放 → 6/1 客戶會撞。）
→ 先從 onboarding / 設定引導拿掉發票相關項。

### 4. 因此教學內容變成
登入 → 改密碼 → 建員工 + 派角色 → （各功能首次使用時的 just-in-time 設定提示）。
乾淨、可委派、不嚇跑新客戶。

### 5. 跟「移除 SetupBanner」一致
不要全域「設定 todos 未完成」牆 → 改「點到點 just-in-time 提示」（在需要某功能時才提示設定）。
