# 出納單手續費 → 團成本：設計決策

> 2026-05-27 William × Claude 討論定案。**狀態：設計定案、尚未實作。**

---

## 問題

- 銀行跨行手續費是公司「為了付這個團的款」真實多花的錢。
- 但它不是「欠供應商的錢」→ 不能進供應商請款單金額（會多付供應商）。
- 現況：手續費只活在出納單上、**沒進團成本** → 結團少算成本 → 團利潤虛高。
- 連帶：**團毛利會餵給員工獎金結算**（`bonus-settlements` 吃團利潤），所以手續費漏算 = 獎金多發。

## 核心決策：單一真相（SSOT），手續費永遠掛在出納單上

> **不為手續費另生任何「請款單 / 出納單」。手續費就是出納單上的一個欄位（`disbursement_order_items.fee_amount`、附帶所屬團）。團成本要算時，直接「讀」這個欄位累加。**

### 為什麼不複製出去（兩種鬼打牆）

1. 若生「pending 手續費請款單」→ 會被下次出納單撈走 → 又生手續費 → 迴圈。
2. 若「請款單 + 出納單一起生」→ 改/刪原始出納單時，衍生紀錄不會即時同步刪 → 孤兒、對帳打架。
- 兩者同病根：**複製一份出去獨立存，源頭一動複本就對不上。**
- 解法：不複製。改/刪出納單 → 手續費自動跟著變（它本來就附在上面）→ 無同步邏輯、無孤兒、無迴圈。

## 兩種收費模式共用同一條流程

| 模式（`workspaces.transfer_fee_mode`） | `fee_amount` 怎麼算 |
|---|---|
| 平均分攤（average / 現行 per-payer） | 實際銀行成本拆出來 |
| 統一收費（unified） | 公司定額 |

- **存法、讀法、顯示完全一樣，唯一差別是金額算法。** 兩個方向收成同一條流程。
- **切換模式不動欄位 / 畫面 / 結構** —— 只是之後新出納單用新算法算。不用改 schema、不用搬資料。
- **切模式只影響之後的單；已出帳舊單維持當時算的值**（歷史事實、不回頭重算）。新舊混存也不亂。

## 顯示

- 團總覽 / 結團頁：多一條「**銀行手續費**」成本行 = 該團出納手續費**累加**。
- 預設只顯示總額；給「展開」鈕（平常不點，對帳對不起來才看明細）。金額小、保持清爽。

## 認列時間點

- 在「**出帳**」當下認列（實際付款、銀行真的扣錢那一刻），不是建出納單草稿時。

## 先擱置（之後再談）

- **會計傳票 / unified 賺差現形**：之後在「傳票 / 報表」層加一塊讀出納單手續費即可，**不影響本流程、不需生請款單**。（William：沒傳票公司也感覺不出收多少，先不碰會計。）
- **獎金追溯**：手續費進團成本 → 團利潤微降 → 獎金微減（變準）。舊團要不要追溯、或只從今以後 → 待 William 拍板。
