## C5. Realtime 訂閱缺口

`createEntityHook` 內建會自動掛 `useRealtimeSync` 訂 Supabase channel、**但前提是 Supabase 端 `supabase_realtime` publication 要有那張表**。

| Entity                | 訂 realtime                             | publication | 結果                  |
| --------------------- | --------------------------------------- | ----------- | --------------------- |
| **todos**             | ✅ 明確標 `cache: high, // Realtime 刷` | ❌ 缺       | 🔴 訂了白訂           |
| **cis_clients**       | ✅ createEntityHook 內建                | ❌ 缺       | 🔴 訂了白訂           |
| **cis_pricing_items** | ✅ 內建                                 | ❌ 缺       | 🔴 訂了白訂           |
| **cis_visits**        | ✅ 內建                                 | ❌ 缺       | 🔴 訂了白訂           |
| channel_messages      | ✅                                      | ✅          | 正常                  |
| 其他 33 tables        | ❌ 無 realtime                          | ✅ 34/34    | 單向（無 push、合理） |

**修法**：對這 4 張表跑：

```sql
ALTER PUBLICATION supabase_realtime ADD TABLE public.todos;
ALTER PUBLICATION supabase_realtime ADD TABLE public.cis_clients;
ALTER PUBLICATION supabase_realtime ADD TABLE public.cis_pricing_items;
ALTER PUBLICATION supabase_realtime ADD TABLE public.cis_visits;
```

**自動偵測**：`scripts/audit-realtime.ts` 已就位、加 `--db` 旗標跑可自動比對。應加進 CI / 動 entity 前 pre-commit。

### Ratchet 維護機制（health report 後續使用方式）

從 Round 4 起、本 health report 改成「**Dashboard 監控檔**」、不再每次大改：

**每月一次**（或新功能上線後）跑：

```bash
# 1. 散刻 mutate 殘留檢測（應 0）
grep -rEn "mutate\\(['\"]" src/ --include="*.ts" --include="*.tsx" | grep -v "/data/core/" | grep -v "/lib/swr/" | grep -v ".test." | wc -l

# 2. SWR rule 違反（baseline 外、應 0）
npm run lint 2>&1 | grep -cE "no-direct-useswr-in-pages|no-direct-supabase-writes"

# 3. Baseline 趨勢
jq -r '[.[] | to_entries | .[].value.count] | add' .eslint-suppressions.json

# 4. apiMutate / invalidateXxx 採用率
grep -rln "apiMutate" src/ | grep -v "node_modules\\|.test.\\|/lib/swr/" | wc -l
grep -rln "invalidate[A-Z][a-zA-Z]*()" src/ | grep -v "node_modules\\|.test.\\|/data/" | wc -l

# 5. Realtime publication 缺口
npm run audit:realtime -- --db
```

數字若退化 → 重做一次健檢、找退化點、修。
數字若進步 → 更新 dashboard 表格。

**從此本健檢檔案進入「monitor」狀態、不再每天大改。**
