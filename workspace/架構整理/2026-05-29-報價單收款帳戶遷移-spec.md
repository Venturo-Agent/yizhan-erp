## 0. TL;DR（最終設計）

1. **bug**：公司設定整份 blanket update `workspaces`，`bank_code` 被載入轉成 `''`（假空白），違反 FK→`ref_banks`。移除公司設定銀行區塊後自然消失（+ 加 null 正規化保險）。
2. **帳戶帶分公司**：`bank_accounts` 加 `branch_id`（NULL = 全公司共用）。修「多分公司帳戶混在一起」的缺口。已實證需要：御風(2 分公司/3 帳戶)、角落(1/2)。
3. **每張報價單記帳戶**：`quotes` 加 `bank_account_id`。開單自動帶該分公司的「報價單帳戶」；**該分公司只有一個 → 不用選**；多個 → 下拉選；選擇記在報價單上、重印一致。
4. **綁報價單顯示旗標**：`bank_accounts` 加 `is_quote_display`（與「可出帳」並列的勾選），控制哪些帳戶可出現在報價單下拉。
5. **schema 缺口補齊**：`bank_accounts` 加「分行 `bank_branch`」「戶名 `account_holder_name`」（報價單要顯示但表沒有）。

> 這版把舊規格的「Phase 2 才做分公司」提前並簡化：不靠資料庫綁死分公司自動配對，而是**開單的人選**（預設帶入、單一免選），分公司維度只是用來「過濾候選 + 預設」。

## 1. 背景與根因（已查證）

### 1.1 bank_code 空白存檔 bug
- 存檔：`src/app/(main)/settings/company/page.tsx:191-211`（`supabase.from('workspaces').update(整份 form)`）。

## 2. 最終架構

### 2.1 核心概念
- 收款帳戶 SSOT = `bank_accounts`（不再是 workspaces.bank_*）。
- 每個帳戶可標記：屬於哪個**分公司**（`branch_id`，NULL=全公司共用）、能否**出帳**（既有 `is_disbursement_eligible`）、能否**出現在報價單**（新 `is_quote_display`）。
- 每張報價單記住自己用哪個帳戶（`quotes.bank_account_id`）。

### 2.2 報價單帳戶解析規則（核心）
```
候選(quote) =
  bank_accounts
  WHERE workspace_id = quote.workspace_id
    AND is_active
    AND is_quote_display
    AND (branch_id = quote.branch_id OR branch_id IS NULL)   -- 自己分公司 + 全公司共用

顯示/儲存：
  1) quote.bank_account_id 已設 → 用它（重印一致）
  2) 否則候選恰好 1 筆 → 自動採用（不跳選擇器；William：「只有一個就不用選」）
  3) 否則候選多筆 → 報價單編輯顯示下拉、選後寫入 quote.bank_account_id
  4) 候選 0 筆 → 報價單顯示提示「請至財務設定設定報價單收款帳戶」
```
- 顯示欄位對應：戶名←`account_holder_name || 公司全名(legal_name)`、銀行←`bank_name`、分行←`bank_branch`、帳號←`account_number`。

> 命名注意：`branch_id` = 公司**分公司**；`bank_branch` = **銀行分行**（如「信義分行」）。兩者不同，勿混。

## 3. Schema 設計

### 3.1 Migration A — bank_accounts（草稿）
```sql
-- supabase/migrations/<ts>_bank_accounts_branch_and_quote_display.sql
-- 為什麼：補分公司維度（多分公司帳戶混在一起）+ 報價單顯示旗標 + 報價單要印但缺的分行/戶名。
BEGIN;
ALTER TABLE public.bank_accounts
  ADD COLUMN IF NOT EXISTS branch_id           uuid REFERENCES public.branches(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS is_quote_display    boolean NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS bank_branch         text,
  ADD COLUMN IF NOT EXISTS account_holder_name text;

CREATE INDEX IF NOT EXISTS idx_bank_accounts_branch ON public.bank_accounts(branch_id);

COMMENT ON COLUMN public.bank_accounts.branch_id           IS '所屬分公司（NULL=全公司共用/總部）';
COMMENT ON COLUMN public.bank_accounts.is_quote_display    IS '是否可出現在報價單收款資訊（解析規則見遷移 spec §2.2）';
COMMENT ON COLUMN public.bank_accounts.bank_branch         IS '銀行分行（報價單顯示用，如「信義分行」）';
COMMENT ON COLUMN public.bank_accounts.account_holder_name IS '戶名（報價單顯示用；空則退回公司全名）';

NOTIFY pgrst, 'reload schema';
COMMIT;
-- rollback:
-- DROP INDEX IF EXISTS idx_bank_accounts_branch;
-- ALTER TABLE public.bank_accounts
--   DROP COLUMN IF EXISTS branch_id, DROP COLUMN IF EXISTS is_quote_display,
--   DROP COLUMN IF EXISTS bank_branch, DROP COLUMN IF EXISTS account_holder_name;
```

### 3.2 Migration B — quotes（草稿）
```sql
-- supabase/migrations/<ts>_quotes_bank_account_id.sql

COMMENT ON COLUMN public.quotes.bank_account_id IS '本報價單顯示的收款帳戶（NULL=依 §2.2 解析候選）';
NOTIFY pgrst, 'reload schema';
COMMIT;
-- rollback:
-- DROP INDEX IF EXISTS idx_quotes_bank_account;

```

### 3.3 Migration C — backfill（草稿，避免上線即掉報價單銀行資訊）
```sql
-- supabase/migrations/<ts>_backfill_quote_display_from_workspaces.sql
-- 為什麼：原報價單讀 workspaces.bank_*；搬家後若無帳戶被勾 is_quote_display 會掉資訊。
-- 策略：每個有 bank_account 填寫的 workspace，挑 is_default 帳戶（無則任一 active）設 is_quote_display=true，
--       並把 workspaces 的分行/戶名補進該帳戶（僅在帳戶該欄為空時）。
BEGIN;
WITH pick AS (
  SELECT DISTINCT ON (ba.workspace_id) ba.id, ba.workspace_id
  FROM public.bank_accounts ba
  WHERE ba.is_active
  ORDER BY ba.workspace_id, ba.is_default DESC, ba.created_at ASC
)
UPDATE public.bank_accounts ba
SET is_quote_display = true,
    bank_branch         = COALESCE(NULLIF(ba.bank_branch, ''), w.bank_branch),
    account_holder_name = COALESCE(NULLIF(ba.account_holder_name, ''), w.bank_account_name)
FROM pick p
JOIN public.workspaces w ON w.id = p.workspace_id
WHERE ba.id = p.id;
COMMIT;
-- 註：多分公司（御風/角落）的 branch_id 維持 NULL（全公司共用），上線後由人工指派各分公司帳戶。
```

### 3.4 workspaces.bank_* 去留
- Phase 1：公司設定 UI 整塊移除（William 已選），DB 欄位**暫留死欄**，觀察期後另開破壞性 migration 清除（附 rollback）。
- bug 根治：`CompanyFormData` + blanket update 不再含 bank_*，FK 違反消失；另加 `'' → null` 正規化保險（防未來新欄再踩）。
