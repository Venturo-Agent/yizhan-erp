## 0. 一句話

把「爬旅行社行程 → 萃景點 → 人工審核 → 匯入 ERP」做成一條**獨立產品線**：專用機 24/7 爬、獨立網站給工讀生審、審好才進 ERP。

---

## 1. 願景（William 構想，2026-05-27 凌晨提出）

- **另一臺閒置電腦當「爬蟲工廠」**：平常沒人操作、不影響工作，可 24/7 持續爬。
- **用便宜 LLM（MiniMax 系 / Llama）大量爬**多家旅行社網站、壓低 token 成本。
- 那臺機器透過 **Claude code** 運作（像 yizhan-erp 這套），加裝 **OpenClaw 的「龍蝦」（= Llama 整合）**；龍蝦用 MiniMax，token 便宜、適合大量爬。
- 爬完 Claude code 負責**回整 + 篩選**，產出**各國家的候選資料庫清單**。
- 清單交由**人力（工讀生／委外）篩選**：看一個獨立頁面就好、不需進 ERP，可幫忙檢查、潤飾內容、上傳圖片。
- 審好的資料**最後彙整進 ERP**，由 Claude code 負責部署。

### 為什麼這概念好（解決今晚實測的兩個痛點）
1. **爬太快會被封**：今晚實測雄獅約爬 80 頁就封我們 IP。專用機 24/7 慢慢爬、本來就不趕、降速也無所謂。
2. **大量候選誰來審**：今晚一輪就產出 1,952 個候選，需要人審。獨立審核站給工讀生、不佔 ERP、不需技術背景。
3. **版權**：人工審核 + 潤飾這一關，正好把競品的「事實」洗成我們自己的內容 → 版權風險再降一層。

## 2. 架構 4 塊

| # | 組件 | 做什麼 | 關鍵考量 |
|---|------|--------|----------|
| 1 | **爬蟲工廠** | 專用機 24/7 爬多家旅行社、Llama/MiniMax + Claude code、產候選清單 | 降速防封、續跑、多站 parser |
| 2 | **資料橋** | 爬蟲機的清單 → 安全送到審核站 | 怎麼傳（GitHub？API？DB？） |
| 3 | **審核站** | 獨立網站、**擋 SEO/Google**、工讀生登入審核/潤飾/上圖 | 登入牆、不被搜尋引擎收錄、非技術人員友善 UI |
| 4 | **匯入閘** | 審好的 → 同步進 ERP 的 attractions / 知識庫 | 一鍵同步 or API；標來源 + 未驗證 |

### 待 William 拍板的設計分歧（影響怎麼蓋）
- **審核站「擋 SEO」**：最穩做法是「要登入才看得到」（不只靠 noindex / robots.txt 叫 Google 別收錄）。建議做登入牆 + 工讀生帳號。
- **審核站 vs ERP 的關係**：(A) 完全獨立小網站、最後用「同步」推進 ERP；或 (B) 其實是 ERP 的一個「隱藏分頁」（呼應 William 想在 LINE bot 設定頁加分頁）。兩條路差很多。

## 3. 今晚已驗證的 pipeline（雄獅旅遊實測 POC，全程未碰 production DB）

完整跑通「爬 → 萃 → 去重 → 候選清單」，數字：

- 爬蟲用**官方 sitemap** 發現全站 **3,555** 個行程種子（不必逆向 API）。
- 實爬 **511 團**（被封後降速重啟、最終 fail=0）。
- 萃取 **464 團**（Claude subagent 萃、5 並行接力）。
- 跨團去重後 **1,952 個唯一景點候選**（✅ 確定新增 1,933 + ⚠️ 待確認 19），涵蓋 **63 國 600 城市**。
- 對照現有 ERP `attractions`（2,544 筆）去重：命中現有 1,247、跳過。

### Pipeline 三階段
1. **爬**：`discover.py`（sitemap 抓 GroupID）→ `scrape.py`（headless chromium 渲染 detail 頁、parser 抽逐日行程原文）→ 存 `raw/<GroupID>.json`。
2. **萃**：Claude subagent 讀 raw 行程原文 → 萃乾淨景點（name/english/country/city/description）→ 對本地索引去重 → 存 `extracted/<GroupID>.json`。
3. **收尾**：合併全部 extracted → 跨團去重 → 產候選清單（readable.md 給人看 + final.json 給匯入）。

## 4. 可複用的技術 know-how（另一臺機器照這個省得重踩）

### 爬蟲
- 雄獅行程詳細頁 URL：`https://travel.liontravel.com/detail?GroupID=<產品代碼>`，也吃 `?NormGroupID=<uuid>`（會 302 補全）。
- 行程頁是 Angular SPA、**必須用無頭瀏覽器渲染**（等 8–15 秒）、純抓 HTML 原始碼拿不到內容。
- **反爬閾值**：約 80 頁/IP 觸發封鎖（網路層 drop TCP、google 正常但雄獅全 timeout）。
- **降速防封**：單線（不要多 worker）+ 每頁隨機 sleep 6–15 秒 ≈ 150–170 頁/小時，實測 fail=0 穩定。
- **chromium 洩漏雷**：每頁 new fetch 會在 timeout 時留孤兒進程、堆積到吃光資源雪崩。修法：整批共用**單一 DynamicSession**（with context manager）、瀏覽器只關一次。
- **熔斷**：連續 8 次硬失敗（timeout/連線拒）就停 + 寫 BLOCKED flag、別無限重試加重封鎖。
- **續跑**：已存在的 raw/<id>.json / .SKIP 秒跳過。
- 被封後**等約 2 小時 IP 自動解封**、降速重啟即可（今晚實測有效）。

### 萃取（LLM 判斷，今晚用 Claude subagent；那臺機器可換 Llama/MiniMax 試）
萃取 prompt 核心規則（已驗證品質好）：
- ✅ 收：觀光地/神社寺廟/城堡/老街/庭園/自然景觀(山湖峽谷海岸瀑布冰河)/博物館/世界遺產/溫泉鄉/市場公園地標
- ❌ 剔：交通工具(纜車小火車遊船徒步航班郵輪)/活動體驗(秀/DIY/SPA/浮潛/祭典)/主題樂園/航班報名條款/購物站(藥妝免稅OUTLET百貨)/純抽象詞/餐廳飯店
- 合併同團內同地點不同寫法（名古屋城/本丸御殿 → 一個）
- description **一律自己改寫事實 30–80 字、嚴禁照抄文案**（版權）
- **country_hint 不可信**（常把出發地/標題誤判）、國別一律以行程內文判斷

### 去重（關鍵把關，避免髒資料）
- 對現有主檔做 name + english_name 模糊比對。
- **國別閘門**：同英文名但不同國 → 不算重複（防「長灘島 Long Beach」帛琉誤中越南、「大峽谷」美西誤中中國）。今晚多次救命。
- **別名要抓**：御蔭橫丁=托福橫丁、湯布院=由布院、賀茂御祖神社=下鴨神社、明洞=首爾明洞、甘川洞彩繪文化村=甘川洞文化村…
- 同國同品牌不同分點 → 標待人工確認、不硬合併。

### 雄獅特有的坑
- 郵輪團/自由行團多半「全日自由活動/自費自理」、無具名景點 → 萃 0、寫空檔。
- **價格沒抓**：今晚 parser 只存逐日行程文字、沒抓整團報價（NT$）。William 已拍板「整團報價當市場參考價」列**下一次任務**（要改 parser 重抓、存行程層級資料、喂 RAG「去X大概多少」；競品售價只當參考非我們報價）。
