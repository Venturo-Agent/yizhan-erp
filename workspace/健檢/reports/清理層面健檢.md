## B. Knip 揭露的 dead code

### B1. 統計
- **134 unused files** — 從未被 import
- **456 unused exports** — 函式 / 常數宣告了但沒人 call
- **322 unused exported types** — TypeScript type 宣告了但沒人 reference
- **9 unused npm dependencies** — package.json 包進來但 import 0 次

### B2. 重大 unused dependencies（猜測、需 verify）
從 knip 報告看、可能包含：
- 某些「曾經要用、後來換掉」的 lib
- dev-only 工具被誤放 dependencies
- 重複功能 lib（譬如多裝 date library）

### B3. 不確定要不要剪的 unused exports
**456 個** unused export 不能一律刪：
- 有些是「給未來模組用」的 helper（可暫留）
- 有些是 type 給外部 module 用的（要看 module export pattern）
- 有些是 dev / debug 用的（不要刪）

**建議**：**knip 結果列在 P1**（不擋 PR、但每月清一次）。

### B4. Duplicate exports（3 個）
從 knip：
```
canManageRoles | hasAdminCapability         — src/app/api/lib/check-capability.ts
formatDateCompact | formatDateCompactPadded — src/lib/utils/format-date/basic.ts
formatDateTW | formatDateDisplay            — src/lib/utils/format-date/basic.ts
```

意思：同檔同函式有兩個 export name、用哪個取決於 import 哪個。
**修法**：保留一個、deprecate 另一個。30 分鐘搞定。

## F. 跟其他 4 維度的關係

- **效能**：清完 unused 後 bundle 小一點（影響輕微）

- **架構**：影響大（搞清楚哪些 module 還活著、哪些半成品、哪些死掉）
- **開發品管**：清理工具（knip）可加進 CI 跑、報告數字越來越小代表健康
