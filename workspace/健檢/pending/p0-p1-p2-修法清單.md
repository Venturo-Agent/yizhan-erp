### P0-1：accounting 全模組補 entity hook + 搬遷

**痛點**：傳票/科目/支票/4個財報/期間結算/開帳餘額**整個會計模組** 7 頁繞 entity hook、無 realtime。
**症狀**：打傳票、建科目、結帳，要 F5 才看到。
**證據**：

- `accounting/vouchers/page.tsx` 直接 `supabase.from('journal_vouchers')`（lint suppress count 0、純讀沒被 rule 抓）
- `accounting/accounts/page.tsx` 直接 `supabase.from('chart_of_accounts')`
- `accounting/checks/page.tsx` 直接讀 + 直接 update（lint suppress count 2）
- `accounting/period-closing/page.tsx` 直接 insert/read
- 4 個財報頁直接 supabase.read

**修法**：

1. 新建 entity hooks：
   - `src/data/entities/journal-vouchers.ts`
   - `src/data/entities/chart-of-accounts.ts`
   - `src/data/entities/checks.ts`
   - `src/data/entities/accounting-period-closings.ts`
2. 從 createEntityHook 產 useJournalVouchers / useAccounts / useChecks / usePeriodClosings 等
3. 每頁改用 entity hook
4. 加 realtime publication（journal_vouchers / chart_of_accounts / checks / accounting_period_closings 4 張表）
5. invalidate 串連：寫入後 `invalidateXxx()`
6. 4 個財報頁的 `chart_of_accounts` + `journal_lines` 直接讀 → 改 entity hook（read-only 不需 invalidate）

**風險**：中。會計模組是核心、改要小心。建議先寫測試、再搬。
**估時**：8-12 小時
**依賴**：需先補 audit:realtime 在 CI、寫完跑驗 publication

### P0-4：把 `audit:writes` + `audit:realtime` 加進 CI（PR 守門）

**痛點**：5/14 trigger × API onboarding 撞號事故是因為沒自動偵測雙寫。同類事故會再發生。
**證據**：

- `audit:writes` / `audit:realtime` 在 package.json 已寫
- `.github/workflows/audit-rls.yml` 只跑 audit:rls
- 紅線 #E 偵測規則無 CI 強制力

**修法**：

1. 改 `.github/workflows/audit-rls.yml` 加 jobs 跑：
   ```yaml
   audit-writes:
     runs-on: ubuntu-latest
     steps:
       - uses: actions/checkout@v4
       - uses: actions/setup-node@v4
       - run: npm ci
       - run: npm run audit:writes
   audit-realtime:
     runs-on: ubuntu-latest
     steps:
       - uses: actions/checkout@v4
       - uses: actions/setup-node@v4
       - run: npm ci
       - run: npm run audit:realtime # code grep only，不需 DB
   ```
2. 不需要 DB secret（grep only mode 就夠 PR 守門）

**風險**：低。純 yaml 變更、不動 code。
**估時**：30 分鐘
**依賴**：無
