一句話總結

全站視覺現在最大的毛病是「一個概念有兩條色軌」：同樣是「成功 / 失敗」，有的地方貼美術色（莫蘭迪綠 / 紅）、有的地方貼語意色（status-success / danger）。在預設莫蘭迪主題這兩條剛好同色、看不出問題；一切到 airtable / 克萊因藍主題就分岔成兩種綠、兩種紅，畫面就花了。其餘是圖示款式、尺寸、按鈕做法各自為政。下面六組請你逐一挑。

✏️ 2. 編輯圖示 ── 4 種「鉛筆」同時在全站使用

同樣是「編輯」這個動作，列表 / 表單 / 設定頁各自挑了不同的鉛筆圖示。下面用真實 lucide SVG 同尺寸（24px）畫出來，下面標全站使用次數。請挑一個當「編輯」唯一圖示。

★ 建議統一成這個
Pencil
全站用 25 次
Edit
全站用 15 次
SquarePen
全站用 6 次

註：lucide 新版把 Edit2 跟 Pencil 都對應到同一支「斜鉛筆」path（看起來一樣）；Edit 跟 SquarePen 都是「方框內鉛筆」（也一樣）。所以實際畫面上是兩種長相、但 code 裡有四個名字在亂用 → 維護時容易誤判。

白話：就像四個櫃台、有的掛「斜鉛筆」牌、有的掛「方框鉛筆」牌，其實都是「改資料」的意思。客戶看不同頁面會覺得「這兩個按鈕是不是不一樣功能？」統一成一個，認知負擔最低。
建議統一 全站「編輯」一律用 Edit2（用最多、最不用改）。其餘 Pencil / Edit / SquarePen 收斂掉。 ← 請確認，或改選你喜歡的那一個

操作欄共用骨架（ActionCell）目前用 0.95em（會隨字級縮放）；共用 Button 內建 size-4（固定 16px）；少數頁面手寫 12 / 14 / 20px。

白話：同一排操作按鈕，有的圖示偏小、有的偏大，眼睛掃過去會覺得「沒對齊」。挑一個尺寸、全站收斂，視覺才整齊。
建議統一 操作欄圖示統一 16px（= w-4 h-4 = lucide 預設）；需要隨字級縮放的場景才用 0.95em。 ← 請確認

操作欄三套做法（這個要收斂）

同樣是列表每一行右邊的「編輯 / 刪除」，三種頁面用三種寫法。並排看差異：

① ActionCell 黃金標準（訂單管理）
高 28px、圖示+文字、灰色 hover 金底；危險走語意紅。← 這套是黃金標準。
② 只剩骨架、沒上色（部分頁面）
有骨架但沒接語意色、刪除沒變紅，跟黃金標準長得不一樣。
③ 認證頁 inline 漸層大按鈕
登入 / 認證頁自己手寫的金漸層大按鈕，跟列表操作欄完全兩種風格。
白話：同樣是「每一行右邊的編輯 / 刪除」，訂單頁長一個樣、別的頁長另一個樣、刪除有的紅有的灰。客戶在不同頁面操作會覺得「這軟體是不是好幾組人各做各的」。
建議統一 全站列表操作欄一律走 ① ActionCell 黃金標準（已是 SSOT，只差把②③收編進來）。 ← 請確認

白話：左邊是公司統一做的柔和標籤（會跟著主題換色）；右邊是有些頁面圖方便、自己貼了「Tailwind 內建的鮮豔紅黃綠藍」。右邊那種一換主題不會變、而且顏色太亮、跟莫蘭迪的素雅調性打架，像穿花襯衫進公司。
建議統一 ①全站狀態標籤只用 共用 StatusBadge，手刻 Tailwind 色全部收編。②順手把 StatusBadge 的 success / danger 從美術色改成語意色（呼應第 1 組）。 ← 請確認這兩件

🚨 6. 真 bug ── 打錯一個字，紅點直接消失

這不是「選哪個」，是「確認要不要修」。AI 助理的未讀紅點，底色 token 被多打了一個「0」（bg-status-danger-bg0），系統裡根本沒這個顏色 → 渲染出來是透明的，紅點等於隱形。並排看實際長相：

❌ 現況（打錯字 = 透明）
AI 對話3
紅底沒出來、數字飄在外面
bg-status-danger-bg0

多了結尾的「0」、CSS 找不到這個 class → 沒套到任何背景色。出現在 AI 對話分頁 / 側邊欄未讀數共 3 處。

✅ 修正後（正確紅底）
AI 對話3
紅底正常、一眼看到未讀
bg-status-danger

拿掉多餘的「0」、改用正確的危險色 token，紅點就上色了。

白話：就像收銀機上「有新訂單」的紅燈，師傅接線時電線標籤多寫一個字、燈泡根本沒通電。客戶有 3 則未讀 AI 訊息，但那顆提醒紅點是透明的、看不到、會漏看訊息。改一個字就好。

同類問題：用到「系統裡不存在的顏色」
