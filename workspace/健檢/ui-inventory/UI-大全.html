🚨 真 Bug 區（救護車式：先講會壞畫面的）

這幾個不是「風格不統一」，是 class 名 / token 名寫錯、瀏覽器解析失敗、樣式靜默掉光——畫面當下就是壞的。修這幾個成本最低、收益最高，建議第一優先。

BUG 1 · 高優先

bg-status-danger-bg0（多打一個 0、未讀紅點沒底色）

AI Hub 未讀數紅點 3 處（主列表 / sidebar 展開態 / sidebar 收合態）都寫 bg-status-danger-bg0。合法 token 是 bg-status-danger（或 -bg），-bg0 不存在 → 紅點根本沒上色。

證據：ai.md L81、L123（譬喻：警報燈裝了、但燈泡型號寫錯、按下去不亮）。

BUG 2 · 最嚴重 / 整頁

/visas 整頁引用「不存在的」morandi 數字階 token

簽證頁的狀態 badge + 表單邊框 + 空狀態文字，全引用 morandi-gray-200/300/500/700、morandi-blue-100/600、morandi-purple-100/600、morandi-green-100/600、morandi-red-100/500/600。但 tokens.css 只定義 morandi-green / morandi-red / morandi-gold（無數字階、無 gray/blue/purple）→ 這些 class 全無對應 CSS、靜默失效。狀態 badge 無底色、表單邊框無色，整頁視覺像「半成品」。

證據：visas.md L27/L32-34/L54/L72/L76（這頁是 91 頁中對齊度最低的一頁）。

BUG 3

/landing 卡片背景 bg-morandi-container/50/50（多一段 /50）

行銷首頁某卡片背景寫成 bg-morandi-container/50/50（多打一段 /50）→ Tailwind 解析失敗、此處實際無背景色生效。

證據：landing.md L75（line 82）。整頁其餘色彩都守住 token、唯這個 typo 是硬 bug。

BUG 4

/app/settings 返回鈕用未定義 class app-header-btn

行動版設定頁返回按鈕 class 寫 app-header-btn，但 globals.css 完全無此定義（已 grep 確認、全專案僅此一處用、其他頁用的是 .app-back-btn）→ 返回鈕掉樣式。同頁空狀態還全用 inline style + 硬編碼 #666。

證據：app__settings.md L25/L62-64。

BUG 5 · 待確認

多頁用疑似不存在的 bg-morandi-background

/no-access、/setup/[token]、/view/[id] 用 bg-morandi-background 當底色，其他頁多用 bg-background / bg-morandi-container / bg-morandi-cream。需確認 morandi-background 是否為有效 token；若無、這幾頁底色也靜默失效。

證據：no-access.md L72、setup__token.md、view__id.md（標「待確認」）。

跨頁變體 ⑤ — 顏色軌（最重要）

這是全站最廣、最根本的不統一：把「美術色」當「語意色」用。

軌寫法問題
美術色當語意色成功用 text-morandi-green、危險/刪除用 text-morandi-red跨 49 個 md 檔出現 morandi-green/red
語意色（正解）成功 status-success、危險 status-danger、警告 status-warning、資訊 status-info跨 57 個 md 檔有用到（多為部分用對）
為什麼是大問題

「預設主題同 hex、但換主題就分岔」

morandi-green 與 status-success 在預設主題下碰巧是同一個 hex，所以肉眼看不出差別、沒人改。但 4 主題體系下，換主題時 status-* 會跟著主題走、morandi-green/red 是固定美術色不跟 → 換主題後「成功是綠、危險是紅」的語意會錯亂。等於現在埋了一顆「換主題才引爆」的雷。

出沒最廣：orders 操作欄（收款 green / 請款·刪除 red）、tours、finance 各頁、workspaces/[id]、3 個認證頁錯誤色（morandi-red）。病根見下方「病根」章節。

🔴 病根（共用層就錯、改一處影響全站）

最值得修的不是各頁、是這兩個「源頭」——它們被全站抄、改源頭等於一次修全站。

病根 1

共用 <StatusBadge> 內部色軌「半套混用」（已坐實原始碼）

所有頁的狀態標籤都 call 這個共用組件。已讀原始碼坐實（src/components/ui/status-badge.tsx:25-32 的 TONE_STYLES）：同一份 map 裡 info/warning 走語意色 status-info/status-warning（✅ 對），但 success=bg-morandi-green/15 text-morandi-green、danger=bg-morandi-red/15 text-morandi-red 卻是美術色（🔴 錯）。亦即組件內部就自相矛盾。→ 所有 caller 的成功/危險 badge 跟著錯。改這 2 行、全站 badge 一次對齊，是「修一處影響全站」的最佳起手點。

病根 2

orders/simple-order-table.tsx（操作欄黃金標準本人）就用美術色

ACTION_BUTTON_BASE 常數的註解明指此檔是「全站列表操作欄黃金標準來源」。但它本身：收款 text-morandi-green、請款/刪除 text-morandi-red、開票 text-morandi-gold —— 把美術色當語意色的做法、從黃金標準被抄到 tours / 各 finance 頁。要全站收斂色軌、得先拍板「改不改黃金標準本人」。

證據：orders.md L98/L104。

結構性 SSOT 問題（不是色、是「該抽沒抽」）

① shared-data master 表複製貼上

shared-data/banks 與 shared-data/countries 是「同一份手刻 <table> 複製貼上」（單一大表 + Input 搜尋 + 啟用欄 ✓/—），沒抽共用「唯讀 master 列表」組件。連表頭背景都三套：bg-muted/50（banks）vs bg-muted/30 text-xs（airports）vs bg-morandi-container/30；行高也三種。啟用狀態用 emoji ✓ 而非 <Badge> status-success，與 insurance-grades 用 Badge 完全不一致。

② 認證頁三套各自手刻

login / change-password / reset-password 是同一族手刻認證頁（同款圓角卡 + 暖金漸層底），但 input/button 的 inline CSS 各寫各的（.login-* vs .cp-* vs .reset-*、漸層配方都不同），錯誤色全用美術色 morandi-red 而非 status-danger。該抽共用認證卡 + 共用認證 input/button。

③ 報表排版兩流派

會計 / 財務報表兩種排版法並存：balance-sheet / income-statement 用 flex justify-between 明細列；general-ledger / trial-balance 用手刻 <table> + bg-morandi-container 表頭。同模組報表互不對齊（會計報表性質、可接受、但宜統一）。
