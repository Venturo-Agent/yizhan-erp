# API 整合 tab — 兩項擴充：教學嵌入 + 額度監控

## TL;DR

每個第三方 integration（OCR / Google Vision / LINE / FB / IG）要在 UI 上補 2 件事：

1. **申請教學嵌入 UI** — 點「設定」展開步驟、員工 / 客戶照做能自己申請、不靠口傳
2. **使用額度顯示** — 用了多少 / 剩多少 / 何時重置、每個 API 不同對應額度

業務目的：**讓公司內部誰接手都能處理**、不靠老員工「我記得怎麼設」。

## 設計（範圍 1：教學嵌入）

### 1.1 Registry 擴 `IntegrationDef`

`src/lib/integrations/registry.ts` 每個 integration 加：

```ts
setupGuide: {
  title: string         // 例：「申請 OCR.space API Key」
  steps: Array<{
    label: string       // 例：「開啟 OCR.space 註冊頁」
    description?: string // 詳細說明
    link?: string       // 對應的 URL（可選）
  }>
  estimatedTime: string // 例：「5 分鐘」
  notes?: string[]      // 注意事項
}
```

### 1.2 各 integration 的教學內容（先寫 5 個）

**OCR.space**：
1. 開 https://ocr.space/ocrapi
2. 點「Free Registration」、填 email
3. 收驗證信、點驗證連結
4. 拿到 API key（Free 版 25,000 次/月、夠用）
5. 回租戶管理 → API 整合 → 護照辨識 → 貼 OCR.space API Key

**Google Vision**：
1. 開 https://console.cloud.google.com/
2. 建 project（命名「venturo-ocr」或客戶名）
3. 左選單 → APIs & Services → Library → 搜「Cloud Vision API」→ 啟用
4. Credentials → Create Credentials → API Key（複製 key）
5. Billing 設定綁信用卡（免費 1000 次/月、超過才收）
6. 回租戶管理 → API 整合 → 護照辨識 → 貼 Google Vision API Keys

**LINE OA**：
1. 開 https://developers.line.biz/console/
2. 建 Provider（如「漫途整合行銷」）
3. 建 Channel → 選「Messaging API」
4. Channel Settings 找：
   - Channel ID
   - Channel Secret
   - Channel Access Token（長期、自己生成）
   - 基本 ID（@xxxx）
5. 貼進「LINE 官方帳號」卡片
6. 設 webhook URL（漫途幫客戶設、走另一個 Logan PR）

**FB Messenger**（Logan M3 規劃中）：待補

**IG**（Logan M4 規劃中）：待補

## 設計（範圍 2：使用額度監控）

### 2.1 Registry 擴 `IntegrationDef`

```ts
usage?: {
  /** 每月免費額度 */
  freeMonthlyQuota?: number
  /** 收費等級對照 */
  pricingNote?: string
  /** Quota 查詢策略 */
  fetchStrategy:
    | { type: 'api'; endpoint: string; parseFn: string }  // 打 third-party API
    | { type: 'db'; column: string }                       // 我們自己 count
    | { type: 'none' }                                     // 無 API、靠 third-party dashboard 看
}
```

### 2.2 各 integration 的額度查法

| Integration | 免費額度 | 查法 |
|---|---|---|
| OCR.space | 25,000 calls / 月 | GET `https://api.ocr.space/parse/imageurl?apikey={key}` 回應 header 有 quota 資訊 |
| Google Vision | 1,000 units / 月 | GCP Monitoring API（要再申請 service account、麻煩） |
| LINE OA | 200 訊息 / 月（免費 plan） | GET `https://api.line.me/v2/bot/message/quota` |
| FB Messenger | 不限（24h 政策） | 無 quota API |
| IG | 不限 | 無 quota API |

⚠️ Google Vision 的 quota 查詢需要 service account、複雜度高、可能先用「客戶自己 GCP dashboard 看、UI 不顯示」這個方案。

### 2.3 我們自己 count

針對 OCR / Vision 這類我們呼叫的、可以在 `api/ocr/passport` 內：
- 加 `ocr_usage_log` 表（workspace_id / integration_code / called_at / cost）
- 每次呼叫寫 1 筆
- UI 顯示「本月已用：X 次 / 免費額度：Y 次」

這個方法更可靠（不用打 third-party API、不會被 rate limit）。

### 2.4 UI 渲染

每張卡顯示：
- 已用 / 額度（progress bar）
- 距離下次重置：N 天
- 接近額度（< 10%）時黃色警告、超過紅色
