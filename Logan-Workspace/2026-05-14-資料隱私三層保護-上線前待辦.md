## 為什麼不能用加密保護整套客戶資料

加密後 = 亂碼、會讓 ERP 99% 功能廢掉：
- 訂單列表搜尋「王小明」→ 搜不到（加密後是亂碼）
- 報表加總團費 → 全廢（金額是亂碼算不了）
- 排序、JOIN、報表 → 全廢

業界沒人這樣做 ERP、會直接不能用。

加密只用在「**敏感小欄位**」：API key / token / 銀行 SWIFT。這套已上線（master key `VENTURO_INTEGRATION_ENCRYPTION_KEY`、AES-256-GCM、2026-05-13 黒羽 + William 拍板設立）。

## Layer 1：技術層（✅ 已有、不用再做）

### 1.1 RLS 隔離

- 每個 workspace 只看自己的資料
- 漫途 admin 用平常帳號也只能看「漫途自己 workspace」、看不到客戶 workspace
- 例外：service_role bypass RLS（理論上漫途能看、要靠 audit log 留紀錄）

### 1.2 Supabase 內建 at-rest 加密

- 磁碟層自動加密（Supabase 出廠帶、不用做）
- 萬一 Vultr 機房硬碟被偷 → 拿到也是加密磁碟、開不了

### 1.3 audit log

- 已寫過 `audit_logs` 表（API route 內 `recordApiAuditContext` 自動寫）
- 紀錄：誰、何時、看了什麼資料、用什麼權限
- ⚠️ 要確認「漫途 admin 用 service_role bypass RLS 也有 log」— Logan 後續 verify

### 1.4 ⚠️ 漫途客服跨 workspace 查 usage log 要 audit（2026-05-14 補）

**現狀漏洞**：
- `GET /api/integrations/usage?workspace_id=X` 可以指定別 workspace
- 漫途有 `workspaces.read` capability、能查任何客戶的 log
- 但**沒寫 audit log**、無法回答客戶「漫途有沒有偷看我的紀錄」

**要補（上線前必做、30 min）**：
1. API route 內偵測 `workspace_id !== ctx.workspace_id`（跨 workspace 查）
2. 寫 1 筆 audit log：誰、何時、查了哪個客戶的 usage log
3. 客戶有權查「漫途查我幾次 / 何時」（給對應 API）
4. 服務條款明文：「漫途為提供技術支援、可查您的 usage log、每次查詢留紀錄供您調閱」

## Layer 2：法律層（❌ 未做、上線前必做）

### 2.1 服務條款 — 資料章節

明文承諾、寫進客戶簽的合約：

> 「漫途員工不會主動讀取貴公司業務資料、除非：
> - 你主動授權（例：協助 onboarding / 客服 ticket）
> - 客服處理你提出的 ticket（會留 audit log、你可調閱）
> - 法律或法院強制要求
> 違反者依民法 184 條 + 個資法 41 條負責。」

### 2.2 隱私權聲明 — 對外公告

放在 venturo.tw 官網 footer、客戶簽約前能看到。內容：
- 蒐集哪些個資（員工 / 客人 / 訂單）
- 用途（旅遊團服務、不轉售）
- 第三方分享（OCR.space / Google Vision / LINE / FB / IG — 客戶自己授權的整合）
- 客戶權利（查閱、修改、刪除、攜出）
- 聯絡窗口（漫途 DPO email）

### 2.3 委外律師 review

- 自己草版 → 律師 review（避免條款無效 / 違法）
- 預算：5000 - 20000 台幣（看複雜度）
- 推薦：找熟 SaaS / 個資法的律師（不是一般民法律師）

### 2.4 Owner + 時程

## Layer 3：認證層（❌ 未做、上線後規模再做）

### 3.1 ISO 27001（資訊安全管理）

- 國際通用、客戶看到貼紙就信
- 成本：5-20 萬台幣 / 年（看公司規模）
- 適合：上線後年營收 > 500 萬時開始準備

### 3.2 SOC 2 Type II（美系常見）

- 美國 / 英國客戶常要求
- 成本：50 萬台幣以上 / 年
- 適合：拓展海外客戶時做

### 3.3 個資法 BS10012 / ISO 27701（隱私專項）

- 比 ISO 27001 更聚焦個資
- 上線 2 年後評估
