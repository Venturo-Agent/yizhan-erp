## Phase 1 — 立即該做（建議下個 session 開頭）

### A. Supabase client 型別統一（🔴 高）

**現況**：13+ 處 `getSupabaseAdminClient() as unknown as SupabaseClient`、type generation 沒涵蓋 inbox_conversations / line_user_profiles / line_conversation_overrides 等 LINE 整合表。

**修法**：
1. 跑 `mcp__supabase-aierp__generate_typescript_types` 重新生 types（5/18 已跑過、但仍有些表沒被 properly typed）
2. 或在 `src/lib/supabase/admin.ts` 加 typed wrapper `getSupabaseAdminClient<T>()`
3. 跨檔批次取代 `as unknown as SupabaseClient` → 直接 typed

**位置**：
- `src/lib/line/debounce.ts` L47, 75, 117, 129, 157（5 處）
- `src/app/api/line/webhook/route.ts` L283, 339, 395, 421 + 9 處
- `src/lib/messaging/inbox.ts` 散落

**為什麼是 🔴**：`as unknown` 繞過 TS 檢查、未來 schema 變動編譯不會擋、會在 runtime 才炸。

### B. 錯誤處理格式統一（🟡 中）

**現況**：
- `profile-client.ts` 全 6 個 fetch fn → 失敗 `return null`
- `push-client.ts` / `reply-client.ts` → 失敗 `return { ok: false, error }`

caller 端要寫兩種判斷、新手容易踩坑。

**修法**：統一改成 `{ ok, data?, error? }` 結構、或全 `null`。建議走 `{ ok, data?, error? }`（更清晰）。
