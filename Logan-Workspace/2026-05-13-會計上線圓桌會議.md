## 二、核心共識（5 souls 全同意）

### 共識 1：訪談 first、明天不上線

**全 5 位同意**：明天**不**直接讓漫途會計上 venturo。先做 1 小時訪談、釐清三件事：
- 她現在每天做什麼？月底結帳前 3 天最痛的事是什麼？
- 她最怕新軟體什麼？（90% 答：報營業稅報錯、字軌錯、跟舊系統對不上）
- 她現在用什麼工具？日常 SOP / 報表格式 / Excel sheets

**Discovery Coach 提供訪談腳本**（Sandler Pain Funnel 三層、60 分鐘）：
- 第 1 層 Surface（20 分鐘）：早上 9 點第一件事 / 上個月最痛的一筆 / 每個月幾號最忙
- 第 2 層 Business Impact（20 分鐘）：如果繼續一年會怎樣 / 跟業務老闆吵架的原因 / 報營業稅最怕什麼
- 第 3 層 Personal Stakes（20 分鐘）：換軟體踩過雷嗎 / 用 venturo 最擔心什麼 / 會被老闆罵嗎

### 共識 2：第一家試用客戶不上會計模組

**4/5 位明確同意（Riley + Alex + Cassandra + Discovery Coach）、Dana 預設立場一致**：
- 第一家試用客戶**不啟用 `accounting` feature flag**
- 賣的是「業務管理 + 收付款追蹤 + 月底匯出 Excel」
- 會計阿姨**繼續用她舊軟體**（鼎新 / 智慧+ / Excel）+ 財政部電子發票平台
- 會計模組當「v2 升級包」、**3-6 個月後**才談啟用

### 共識 3：發票 / 5% 稅不做、走財政部平台、包成 sales feature

**全 5 位同意**：「不做發票模組」**是 product positioning、不是 bug**。

Pitch 公式（Alex + Riley + Discovery Coach 一致版本）：
> 「venturo 不重複造輪子。發票走政府的『財政部電子發票平台』、免費、合規、永遠不會被罰錢。venturo 把訂單 / 收款 / 損益做到最好、發票交給政府專家。每個月對一次帳、5 分鐘搞定。」

**競品比較話術**：
- 「鼎新有發票模組」→「鼎新是傳統 ERP、發票是必備、因為他們也做製造業。我們專做旅行社、發票不是旅行社的核心痛點、所以我們不做、把研發資源放在妳更痛的地方。」

**過濾不適合客戶**：客戶老闆說「我就是要 venturo 開發票」→ 這家不是 P1、禮貌道別。

### 共識 4：CRITICAL 分批做、不是「全修完才上線」

**5 位都同意「分批」、但分批方式各有微調**。最大公約數：

| 批次 | 內容 | 時程 | 為什麼 |
|---|---|---|---|
| **P0 — 上線前必修** | CRITICAL #1 撞號 + #2 silent fail + #4 結團鎖定 | Week 1-2 | 不修漫途自己跑不下去 / 第一家客戶簽不下來 |
| **P0.5 — 並行修（Riley 強烈主張）** | 損益表 TourPnLTab（簡易版聚合、非審計口徑） | Week 1-2 | 老闆每月看不到自家損益 = 3 個月後不續訂 |
| **P1 — 試用期間補** | CRITICAL #3 月結鎖定 + #5 請款雙簽 + AR 管理頁 + 客戶對帳單匯出 + 期初餘額個別明細 | Month 1（簽約後 30 天內） | 客戶第一個月還沒到月結、有時間補 |
| **P2 — 訪談決定** | 訂金 / 尾款 / 多幣別 / 訂單退單 | Month 1-2 | Discovery Coach 強烈：先訪談、確認是否是該客戶 P1 痛點 |
| **P3 — 永遠不做** | 發票 / 開立發票 / 字軌管理 / 銀行對帳 API 整合 | — | Product positioning：走外部 SSOT、不重複造 |

### 共識 5：90 天 onboarding 階梯、用「使用訊號」當 milestone、不是「功能完成度」

**4/5 位（Dana + Cassandra + Riley + Alex）強烈同意**。Discovery Coach 補充「信任曲線」框架。最大公約數階梯：

```
Day 1-7（信任建立）：
  能做：訪談 + 業務員開單試打
  不能：會計模組 / 月結 / 寫入帳
  成功訊號：業務員主動進 ≥ 3 次/天、會計願意看介面
  失敗訊號：業務員 3 天還在問「按哪裡」、會計避開不開

Day 8-30（信任驗證）：
  能做：業務日常（開單 / 請款 / 收款）+ 會計只讀
  不能：會計寫入 / 月結 / 結轉
  成功訊號：業務員 90% 自助、會計能撈出當月應收清單
  失敗訊號：業務員一直找客服、會計撈不到她要的數字

Day 31-60（信任鞏固）：
  能做：會計開始寫入（傳票 / 沖銷）+ 跟舊系統並行
  不能：月結鎖定 / 完全關舊系統
  成功訊號：月底兩邊帳差異 < 1%、會計能跑完月結 checklist 50%
  失敗訊號：兩邊差異 > 5%、會計每天說「不對」

Day 61-90（信任投資）：
  能做：第一次完整月結 in venturo、舊系統當保底
  不能：關閉舊系統
  成功訊號：venturo 月結跑完、差異 < 0.1%、會計簽字認可
  失敗訊號：月結卡在某科目對不上、超過 5 個工作天沒解

Day 91+（完全切換、但保留 Excel 邊角業務）：
  能做：純 venturo、舊系統 read-only
  失敗訊號：第二次月結還卡、要回頭看哪步沒做好
```

**Cassandra 額外條件**：90 天 onboarding 真正成功的訊號是 **「7/15 第一次報稅順利 + 9/15 第二次報稅順利」**、必須跨兩個雙月報稅週。

### 共識 6：並行運作不訂硬期限、以「對帳差異 < 1%」為切換條件

**Dana + Alex + Discovery Coach 一致**、Riley 同意但建議「3 個月硬上限」、Cassandra 同意但建議「至少 2 次報稅週 / 建議 4 次報稅週」。

最大公約數：
- 第 1-3 個月：100% 並行（會計阿姨主跑 Excel、venturo 是輔助）
- 第 4-6 個月：venturo 主、Excel 輔
- 第 7-12 個月：venturo 主、Excel 留特定 sheet
- **不承諾「3 個月切換」**、客戶決定節奏、廠商不逼

**切換 GO 條件**（4 條件全達標）：
1. 連續 2 個月雙邊對帳差異 < 1%
2. 老闆主動說「我只看 venturo 就好」
3. 會計阿姨主動說「舊系統我不用了」
4. 上一次月結 venturo 走完整流程無回退

## 八、補充：月結鎖 / 結團鎖實作精神（William 2026-05-13 12:08 拍板）

圓桌結束後、跟 William 對話釐清「月結鎖 / 結團鎖」要怎麼做、他拍板了：

> **「過了就是過了、要改要在當期開補正交易、不准回頭改」**

這對應會計準則本身的「**前期損益調整**」原則 — 跨期事項一律做在當期、不能回頭改舊資料。

### 關鍵紀律（寫進 CLAUDE.md 紅線 D）

**不准做的事**：
- ❌ 月結後該期間能解鎖（不管是 admin / 平台管理 / 任何 capability）
- ❌ 結團後能重開團（沒有 `reopenTour` / `unlockTour` API）
- ❌ 已確認的傳票能改數字
- ❌ 任何 `forceUnlock` / `adminOverride` / `bypassPeriodLock` 命名的函式

**正確做法**：
- ✅ 月結後發現前期錯 → 在當月開「**沖正交易**」（負金額抵錯的 + 正金額補對的）
- ✅ 結團後發現錯 → 在當期 / 下期開新交易補正、不動原團
- ✅ 跨團調整走「**對沖**」（用 `transferred_pair_id` 雙負雙正、不改原紀錄）
- ✅ 傳票寫錯 → 開「**沖正傳票**」(reverse voucher) 抵銷、原傳票留著當證據

### 業務語意（William 原話）

- 「月結了就是月結了、改的話要在下個月做」
- 「結團了就是結團了、要改的話放到下個月、做下個月的帳」
- 「能解鎖 = 公司在作弊、稅務 / 審計都會炸」

### 技術上反而更簡單

不用做：解鎖機制 / 重開團 API / 獎金重算邏輯 / admin capability

只要做：
- DB trigger：已結帳期間寫入直接 reject
- DB trigger：已結團相關寫入直接 reject
- 錯誤訊息提示「請走補正交易、本期錯誤請在當期開沖正」

### 完整紅線見 CLAUDE.md 紅線 D「不准開作弊後門」
