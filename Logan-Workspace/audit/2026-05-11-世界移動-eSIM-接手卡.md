## 四、技術設計（已對齊 workspace_integrations 平台）

### workspace_integrations 加新 integration entry

`src/lib/integrations/registry.ts` 加：

```ts
{
  code: 'worldmove_esim',
  name: '世界移動 eSIM',
  description: '...',
  affects: ['旅遊團 eSIM 商品 / 訂單 / callback / 流量查詢'],
  fields: [
    { key: 'merchant_id', label: 'Merchant ID', sensitive: false, required: true },
    { key: 'dept_id', label: 'Dept ID', sensitive: false, required: true },
    { key: 'token', label: 'API Token', type: 'password', sensitive: true, required: true },
    { key: 'environment', label: '環境', type: 'text', required: false, placeholder: 'prod / test' },
  ],
}
```

→ 每個 workspace 一筆 row、各自的 DeptId / Token、共用底層 platform

### 5 個新 DB 表（依黒羽卡 5.1）

- `worldmove_products`：商品快取（每週 sync）
- `worldmove_orders`：訂單主表（含 workspace_id 分租戶）
- `worldmove_items`：細項（兌換碼 / QRcode）
- `worldmove_callbacks`：callback 事件 log
- `worldmove_usage_snapshots`：流量查詢快取

全部加 RLS workspace_id 守門。

### API client 結構

```
src/lib/worldmove/
├── client.ts        # axios + 簽章 helper
├── signing.ts       # SHA-1 + 各 endpoint encStr 公式（單測必做）
├── endpoints/
│   ├── quote.ts        # myQueryAll
│   ├── esim-buy.ts     # mybuyesim + querybuyesim
│   ├── esim-redemption.ts
│   └── ...
└── types.ts
```

### Callback handler

`src/app/api/worldmove/callback/[endpoint]/route.ts`

每個 handler 步驟：
1. 驗 encStr 簽章（不通過 401、仍記 log）
2. 寫 worldmove_callbacks
3. 用 order_id 找 worldmove_orders 更新狀態
4. 解析 items 寫 worldmove_items
5. **回 `"1"` 字串**（不是 JSON、踩雷點）
6. 異常 throw → 500 → 世界移動 retry

### Capability + Feature flag seed

DB migration 要：
- workspace_features 預設 row：`worldmove_esim` 給漫途 workspace、預設 false（user 主動開）
- role_capabilities seed：`worldmove.esim.read/write` 給「業務」「助理」role 預設 enabled=true

## 八、關鍵踩雷點（避免重蹈覆轍）

1. **encStr 拼接公式**：每個 endpoint 不同、不是統一 hash params。組錯一律 401。要照 PDF 一個個對、寫單測驗證。

2. **callback 回應**：要回 **`"1"` 字串**（純字串、不是 JSON `{"code":1}`）。回錯世界移動會 retry 3-4 次。

3. **leSIM 分流**：
   - `leSIM=true` 世界移動自家 → 功能完整
   - `leSIM=false` 當地供應商成品卡 → 不支援 2.4 下單並兌換、無法查 usage
   - 業務邏輯必須分流、不能一視同仁

4. **限速**：商品報價週查一次、查單類不能每分鐘戳、流量更新本身 2-3 小時才動

5. **Token per Dept 還是 per Merchant**：影響 workspace_integrations 設計、要先確認

6. **回應碼**：0 成功 / 401 簽章 / 408 eSIM 未產出 / 805 庫存不足 / 808 卡號重複 / 996 凍結 / 999 失敗（其他見 PDF）
